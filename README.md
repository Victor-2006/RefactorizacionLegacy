# **Práctica: Limpieza de Primavera (Refactorización)**

Bienvenido/a al repositorio base de la práctica evaluativa de refactorización del módulo **Entornos de Desarrollo**.

## **Tu Misión**

En este repositorio encontrarás la clase FacturacionLegacy.java. Tu objetivo es utilizar las herramientas de tu IDE (Eclipse, IntelliJ o NetBeans) para limpiar este código aplicando la **Regla del Boy Scout**, eliminando números mágicos y reduciendo la complejidad ciclomática mediante cláusulas de guarda.

## **La Red de Seguridad**

También encontrarás la clase FacturacionLegacyTest.java. Contiene pruebas unitarias que validan el comportamiento del código de facturación.

**Reglas de oro:**

1. Ejecuta los tests **antes** de tocar nada. Deberían estar en verde.
2. **NO modifiques** el archivo de tests bajo ningún concepto.
3. Ejecuta los tests después de cada pequeño cambio que hagas. Si se ponen en rojo, has roto la lógica del negocio. ¡Deshaz el cambio usando Git\!

¡Mucha suerte y a limpiar código\!
